Search Behaviour

i
How long Nudgarr waits before searching the same item again. If your run interval is 6 hours and cooldown is 4 hours, the same item will be searched every single sweep because it's always past cooldown by the time the next run fires. Most indexers recommend waiting at least 24-48 hours between searches for the same item to avoid triggering rate limits or bans.
Minimum hours before the same movie or episode can be searched again. 0 disables.

Maximum Cutoff Unmet movies searched per instance per run. 0 disables.

Throttling

i
How many search commands are sent to your indexer at once per batch. A batch size of 1 means each item is searched individually with a pause in between. Higher values send multiple searches simultaneously which can overwhelm indexers and trigger rate limiting. Keep this at 1 unless you have a specific reason to increase it.
Number of items sent per search command. Smaller values are easier on your indexers.
i
How long Nudgarr pauses between each batch of searches. Combined with batch size this controls the overall pace of a sweep. A sleep of 5 seconds with a batch size of 1 means one search every 5 seconds. Lowering this too much reduces the breathing room between searches and increases the risk of hitting indexer rate limits.
Pause between batches in seconds. Gives your indexers time to breathe.
i
Adds a random delay on top of the sleep time between batches. If sleep is 5 seconds and jitter is 2 seconds, actual pauses will vary between 5 and 7 seconds. This randomness makes Nudgarr's search pattern less predictable and helps avoid triggering automated rate limit detection that looks for suspiciously regular request intervals.
Random extra pause on top of Sleep Seconds to help avoid indexer rate limiting.

⚠️ INDEXER RATE LIMITS

Nudgarr instructs instances to search on your behalf. Start with low limits and a sensible cooldown. Indexers may rate limit or ban if hit too hard. Please be respectful of their limits when using this app.

Backlog Nudges

Searches movies and episodes identified as missing. These searches are on top of your Max Per Run caps.

Radarr

Disabled
i
How many missing movies are searched per Radarr instance each sweep when backlog nudges are enabled. Unlike Cutoff Unmet searches which target items you already have at a lower quality, missing searches target items you have never downloaded. These can add up fast — if you have hundreds of missing movies and set this high combined with a short run interval you can generate a very large number of searches in a short time. Start at 1 and increase slowly.
Maximum missing movies searched per instance per run.
i
Only search for missing items that were added to your library at least this many days ago. This prevents Nudgarr from immediately searching for something you just added and are still expecting to arrive naturally through your RSS feed. Setting this to 0 disables the filter entirely and makes all missing items immediately eligible regardless of when they were added.
Only search missing items added more than this many days ago.

Sonarr

Disabled
i
How many missing episodes are searched per Sonarr instance each sweep when backlog nudges are enabled. Unlike Cutoff Unmet searches which target episodes you already have at a lower quality, missing searches target episodes you have never downloaded. TV libraries tend to have far more missing episodes than movies — a single incomplete series can have hundreds of missing entries. Start at 1 and increase very slowly while watching your indexer's rate limits carefully.
Maximum missing episodes searched per instance per run.

USE WITH CAUTION

Setting Missing Added Days to 0 disables the age filter — all missing items become eligible immediately. Combined with a high Missing Max and short run interval this can generate a very large number of searches in a short time, risking indexer rate limiting or bans. Nudgarr is not responsible for bans resulting from user-configured search behaviour.

⚠️ Newest Added is active and Radarr backlog nudges are enabled. Items closest to your Missing Added Days cutoff will be searched first.

Data Retention

i
How many days Nudgarr keeps history and stats entries before pruning them on the next sweep. This applies to both the History tab and the Stats tab. Lifetime Movies and Shows totals are never affected by pruning — only the individual entry records are removed. Setting to 0 disables pruning entirely and entries are kept forever.
Prunes history and stats entries older than this. Lifetime totals are not affected. 0 disables.

Stats

i
How often Nudgarr polls Radarr and Sonarr for import events on previously searched items. Setting this too low may miss slow downloads — 120 minutes is a safe default. Confirmed imports appear in the Stats tab.
How often Nudgarr checks for confirmed imports.

What's New in v2.9.0

Here's what changed since your last visit.

🔑 API Key Masking v2.9.0
API keys are no longer returned in full from the server. The UI shows the last 4 characters only — the full key is never sent back to the browser after saving.
🔒 Security Hardening v2.9.0
URL validation on test endpoints, Origin/Referer CSRF protection on all POST routes, security response headers, and persistent session key surviving container restarts.
🔌 Instance Modal v2.9.0
Connection test on modal save now fires against the values you just typed — a bad key goes red immediately without needing to hit Save Changes. API Key label now indicates when a key will be masked after saving.
