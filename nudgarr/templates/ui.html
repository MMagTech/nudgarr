What's New in v5.0

v5 is a full UI redesign. Everything works the same way — this is what moved and where.

Tabs Replaced with a Sidebar
The horizontal tab bar is gone. Navigation is now a sidebar with Monitor, Configure, and System groups. Overrides and Filters sit as sub-items.
History, Imports, CF Score, and Exclusions Are Now Library
Four separate tabs are now one Library panel with a view switcher at the top. Exclusions get their own dedicated view with filtering, search, and pagination.
Pipeline Config Moved to Pipelines
Backlog and CF Score settings moved from Advanced and Settings into the Pipelines panel. Settings now covers Schedule, Sweep Controls, Throttling, and Auto-Exclusion only.
CF Score Per-App and Per-Instance Controls
CF Score can now be toggled independently for Radarr and Sonarr. Per-instance overrides add a CF Score enable/disable toggle and sample mode per instance.
Unsaved Changes Indicator
A small amber dot appears next to any nav item with unsaved changes. You can navigate freely between panels without losing pending edits.
Mobile Sidebar Drawer
On narrow screens the sidebar opens as a slide-in drawer from a hamburger button in the topbar. All panels are accessible on mobile — no separate layout.
No Migration Required
No backend changes. Your config, history, cooldown records, exclusions, imports, Intel data, and CF Score index are all untouched. The upgrade is a container restart.

Per-Instance Overrides — How it works

Read this once — it won't show again.

🌐 Global values are the starting point
Every instance starts from the global values set in Settings and Pipelines. Override any field on any instance independently — unset fields always fall back to global.
⚠️ Search volume multiplies per instance
Each instance runs independently. 4 instances × max 15 = up to 60 searches per sweep. Review your indexer's rate limits before raising per-instance caps.
🎯 Nothing changes until you hit Apply
Enabling overrides doesn't change any behaviour on its own. Each instance card has its own Apply button — only saved cards take effect.
↻ Disabling overrides reverts all instances to global immediately
All per-instance values revert to global the moment you disable overrides. Your override values are preserved and restored if you re-enable.
▶ Backlog can be toggled per instance
When overrides are active, each instance can have Backlog enabled or disabled independently, regardless of the global Pipelines setting.
