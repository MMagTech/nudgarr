Glossary

Key terms used in the Nudgarr UI and documentation.

Apprise A Python library that provides a unified interface to over 80 push notification services. Nudgarr uses Apprise to deliver notifications via a single URL configuration. See Notifications.

Auto-Exclusion A title that Nudgarr has automatically added to the exclusions list after its search count reached the configured threshold with no confirmed import. Auto-excluded entries are stored with source=auto and display an amber badge in the desktop Exclusions tab and amber title text on mobile. They can be removed individually from the Exclusions tab or cleared in bulk from the Danger Zone. Distinct from a manual exclusion, which is added explicitly by the user. Configure thresholds per app in Advanced → page 2 (desktop) or the Settings tab (mobile).

Auto-Unexclude The automatic removal of auto-excluded titles after a configured number of days. Runs at the start of each sweep. When a title is auto-unexcluded its search count resets to 0, giving it a genuine fresh start. Only affects entries with source=auto — manual exclusions are never touched. Configured via the Unexclude Days fields in Advanced (desktop) or Settings (mobile). 0 disables auto-unexclude for that app.

Backlog Nudge A search triggered for a monitored item that has never been successfully grabbed (i.e. it is missing entirely, not just below quality cutoff). Controlled by the Radarr Backlog and Sonarr Backlog toggles. Appears as Backlog in the History tab.

Batch Size The number of items included in a single search API call to Radarr or Sonarr. Larger batches reduce the number of API calls per sweep but may be more likely to trigger indexer rate limits.

Cooldown The minimum number of hours Nudgarr waits before searching the same item again. After an item is searched, it becomes ineligible for re-searching until the cooldown window expires. Default is 48 hours.

Cutoff Unmet A Radarr/Sonarr concept — an item that has a file but that file does not meet the quality cutoff defined in its quality profile. For example, a movie grabbed as 720p when your profile requires 1080p. Nudgarr's primary purpose is to trigger fresh searches for these items. Appears as Cutoff in the History tab.

Eligible An item that has passed all filters (not excluded, not queued, not on cooldown) and is available to be picked for searching in a given sweep.

Exclusion A title that has been explicitly removed from all future Nudgarr searches. Exclusions are matched by title (case-insensitive) and apply globally across all instances and both Radarr and Sonarr.

Import A confirmed successful download and import of a searched item. Nudgarr tracks imports via the import check loop, which polls arr apps for recently imported files and matches them against previously searched items.

Import Check Interval How often (in minutes) the import check loop runs. Runs on its own independent timer — not tied to the sweep schedule. Default is 120 minutes.

Instance A single Radarr or Sonarr installation configured in Nudgarr. You can add multiple instances — for example, a standard and a 4K Radarr. Each instance is processed independently and has its own health indicator.

Instance Health A per-instance indicator showing whether Nudgarr was able to successfully communicate with that instance during the last sweep. A red indicator means the last API call to that instance failed.

Jitter A random additional delay added on top of the sleep between batches. Prevents Nudgarr from making API calls at perfectly predictable intervals, which can help avoid indexer rate limiting.

Max per Run The maximum number of items Nudgarr will search in a single sweep for a given type. Max Movies per Run applies to Cutoff Unmet movies; Max Episodes per Run applies to Cutoff Unmet episodes. Missing Max applies separately to Backlog searches.

Missing Added Days A Backlog Nudge filter for Radarr only. Only movies added to the library at least this many days ago are eligible for backlog searches. Gives your RSS feed a window to grab new releases naturally before Nudgarr steps in. 0 disables the filter. This setting does not apply to Sonarr — all missing episodes are eligible regardless of age.

On Cooldown Items skipped during a sweep because they were searched too recently (within the cooldown window). Displayed as a stat on the Sweep tab.

Queued Items already present in the Radarr or Sonarr download queue at the time of a sweep. Queued items are silently skipped and do not consume a search slot — the maximum per run is always filled from actionable items only.

Run Now A button in the UI (and on the mobile home tab) that triggers an immediate sweep regardless of the cron schedule.

Sample Mode The ordering strategy used to pick items for searching. Options are: Random, Alphabetical, Oldest Added, Newest Added. Applied independently to Radarr and Sonarr.

Skip Queued The behaviour of silently bypassing items already in the download queue. Always on — no toggle or configuration required.

Sleep A fixed wait (in seconds) between batches during a sweep. Combined with Jitter to pace API calls and reduce indexer pressure.

State Retention The number of days Nudgarr keeps history and cooldown records. Entries older than this threshold are pruned automatically. Default is 180 days. Set to 0 to disable pruning.

Sweep One full pass of Nudgarr across all enabled instances — fetching Cutoff Unmet lists, applying filters, and triggering searches. Sweeps run on the cron schedule or on demand.

Turnaround The time between the first search Nudgarr triggered for an item (first_searched_ts) and the confirmed import of that item. Displayed in the Imports tab.
