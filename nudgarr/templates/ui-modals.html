What's New in v4.2.0

Here's what changed since your last visit.

CF Score Scan
A third independent search pipeline that finds any monitored file where the custom format score is below the quality profile cutoff, regardless of quality tier. Radarr and Sonarr never surface these through their normal wanted system. Enable in Advanced and configure search limits in the new CF Score tab.
Intel Tab
A lifetime performance dashboard tracking search health, instance efficiency, stuck items, exclusion calibration, library age vs success rate, and quality upgrade paths. Gets richer the longer Nudgarr runs. Reset from the bottom of the Intel tab if needed.
Sweep Tab Redesign
The Sweep tab is rebuilt around three pipeline cards (Cutoff Unmet, Backlog, CF Score), each showing aggregate totals and per-instance breakdowns. A summary row covers Sweep Health, Last Sweep timing, and Imports Confirmed. A full-width feed logs every item searched in the current sweep with its pipeline badge and pagination.
Also in this release
Maintenance Window: suppress scheduled sweeps during a defined time window. Configure in Settings. Manual Run Now always bypasses it.
Backlog Sample Mode Split: Cutoff Unmet and Backlog now have independent sample modes. Grace Period also available in Advanced.
Settings cleanup: Cooldown moved into the Scheduler card. Cutoff Unmet has the full right card to itself.

No Instances Configured

Add a Radarr or Sonarr instance to get started. The Sweep tab will populate automatically once an instance is added.

Per-Instance Overrides — How it works

Read this once — it won't show again.

🌐 Global values are the starting point
Override any field on any card independently. Unset fields always fall back to global.
⚠️ Search volume multiplies per instance
Each instance runs independently. 4 instances × max 15 = up to 60 searches per sweep. Check your indexer before raising any per-instance caps.
🎯 Nothing changes until you hit Apply
Enabling this tab doesn't change any behaviour. Overrides only take effect when you explicitly Apply them on a card.
🔄 Turning this off reverts all instances to global immediately, cooldowns included
Override values are preserved and restored if you re-enable.
📋 Each instance can have backlog on or off
The global backlog toggle no longer controls everything when overrides are active.
