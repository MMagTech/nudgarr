What's New in v4.1.0

Here's what changed since your last visit.

🚫 Auto-Exclusion
Disabled by default. Titles that keep getting searched with no confirmed download can now exclude themselves automatically. Set a search count threshold per app in Advanced → page 2. Auto-excluded titles show an amber badge in the Exclusions tab and can be cleared individually or in bulk from the Danger Zone. A configurable unexclude timer returns them to eligibility after a set number of days.
📊 Import Stats Period
The Imports tab stats card now lets you switch between Lifetime, Last 30 Days, and Last 7 Days. Lifetime totals are protected and survive a Clear Stats — the rolling windows calculate from your import history and update immediately when you switch.
🔧 Under the Hood
Large libraries now get full results — the 500-item pagination cap is removed, so Oldest Added and Newest Added modes work correctly regardless of library size. Log timestamps use container local time. All static files are cache-busted on upgrade so browsers automatically load fresh JS and CSS without a hard refresh. Flask development server replaced with Waitress for more reliable request handling. In-progress sweeps now complete cleanly on container stop.

No Instances Configured

Add a Radarr or Sonarr instance to get started. The Sweep tab will populate automatically once an instance is added.

Per-Instance Overrides — How it works

Read this once — it won't show again.

🌐 Global values are the starting point
Override any field on any card independently. Unset fields always fall back to global.
⚠️ Search volume multiplies per instance
Each instance runs independently. 4 instances × max 15 = up to 60 searches per sweep. Check your indexer before raising any per-instance caps.
🎯 Nothing changes until you hit Apply
Enabling this tab doesn't change any behaviour. Overrides only take effect when you explicitly Apply them on a card.
🔄 Turning this off reverts all instances to global immediately, cooldowns included
Override values are preserved and restored if you re-enable.
📋 Each instance can have backlog on or off
The global backlog toggle no longer controls everything when overrides are active.
