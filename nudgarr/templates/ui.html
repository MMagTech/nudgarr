Throttling

i
How many search commands are sent to your indexer at once per batch. A batch size of 1 means each item is searched individually with a pause in between. Higher values send multiple searches simultaneously which can overwhelm indexers and trigger rate limiting. Keep this at 1 unless you have a specific reason to increase it.
Number of items sent per search command. Smaller values are easier on your indexers.
i
How long Nudgarr pauses between each batch of searches. Combined with batch size this controls the overall pace of a sweep. A sleep of 5 seconds with a batch size of 1 means one search every 5 seconds. Lowering this too much reduces the breathing room between searches and increases the risk of hitting indexer rate limits.
Pause between batches in seconds. Gives your indexers time to breathe.
i
Adds a random delay on top of the sleep time between batches. If sleep is 5 seconds and jitter is 2 seconds, actual pauses will vary between 5 and 7 seconds. This randomness makes Nudgarr's search pattern less predictable and helps avoid triggering automated rate limit detection that looks for suspiciously regular request intervals.
Random extra pause on top of Sleep Seconds to help avoid indexer rate limiting

⚠️ INDEXER RATE LIMITS

Nudgarr instructs instances to search on your behalf. Start with low limits and a sensible cooldown. Indexers may rate limit or ban if hit too hard. Please be respectful of their limits.

Backlog Nudges

Finds and searches items missing from your library on top of your regular cutoff sweeps.

Radarr

Disabled
i
How many missing movies are searched per Radarr instance each sweep when backlog nudges are enabled. Unlike Cutoff Unmet searches which target items you already have at a lower quality, missing searches target items you have never downloaded. These can add up fast — if you have hundreds of missing movies and set this high combined with a frequent cron schedule you can generate a very large number of searches in a short time. Start at 1 and increase slowly.
Maximum missing movies searched per instance per run
i
Only search for missing items that were added to your library at least this many days ago. This prevents Nudgarr from immediately searching for something you just added and are still expecting to arrive naturally through your RSS feed. Set to 0 to remove the age filter and search all missing items regardless of when they were added.
Only search missing items older than this many days. 0 = no age filter.

i
How many missing episodes are searched per Sonarr instance each sweep when backlog nudges are enabled. Unlike Cutoff Unmet searches which target episodes you already have at a lower quality, missing searches target episodes you have never downloaded. TV libraries tend to have far more missing episodes than movies — a single incomplete series can have hundreds of missing entries. Start at 1 and increase very slowly while watching your indexer's rate limits carefully.

What's New in v4.0.0

Here's what changed since your last visit.

🎨 Refreshed UI
The interface has a new look — deeper backgrounds, refined typography using Outfit and JetBrains Mono, and redesigned sweep cards. The instance modal now has a Test connection button — check your URL and API key before applying.
🔍 Quality Upgrade Tracking
Each import now records the quality path — what the file was before the search and what it upgraded to. Find it in the Imports tab as a new Upgrade column with a hover tooltip showing the full upgrade history per item.
🏷️ Tag & Quality Profile Filters
A new Filters tab lets you exclude items from sweep by tag or quality profile, configured per instance. Items matching an excluded tag or profile are skipped before cooldown runs — they never consume a search slot. Load tags and profiles live from each instance, select what to exclude, and Apply. Available in the desktop Filters tab and the landscape Filters nav tab on mobile.
📱 Mobile Redesign
Completely rebuilt mobile experience. Portrait gets a proper four-tab layout (Home, Sweep, History, Settings), instance health and enable/disable on the Home screen, a unified History tab with search history, exclusions, and add-from-history in one place. Landscape gets dedicated Backlog, Execution, Overrides, and Filters tabs.
🔧 Under the Hood
Structured logging throughout — all output goes to stdout and a rotating log file at /config/logs/nudgarr.log (5 MB per file, 3 backups, 20 MB total cap). Log level is configurable live from the Advanced tab. Sweep and import writes are now batched into single transactions. The frontend is now split across dedicated static files (one per domain) that your browser can cache independently, and the backend routes are organised into focused blueprints — each file covers one area. Contributor docstrings added throughout so the codebase is easier to navigate.

Per-Instance Overrides — How it works

Read this once — it won't show again.

🌐 Global values are the starting point
Override any field on any card independently. Unset fields always fall back to global.
⚠️ Search volume multiplies per instance
Each instance runs independently. 4 instances × max 15 = up to 60 searches per sweep. Check your indexer before raising any per-instance caps.
🎯 Nothing changes until you hit Apply
Enabling this tab doesn't change any behaviour. Overrides only take effect when you explicitly Apply them on a card.
🔄 Turning this off reverts all instances to global immediately, cooldowns included
Override values are preserved and restored if you re-enable.
📋 Each instance can have backlog on or off
The global backlog toggle no longer controls everything when overrides are active.
